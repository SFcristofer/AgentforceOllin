PLAN DE IMPLEMENTACIÓN PROFESIONAL: BOT API OLLIN PROTECCIÓN EN AGENTFORCE
================================================================================

Este documento detalla todas las funciones, reglas de negocio y endpoints especificados 
en la "Guía de Integración — Bot API" para su implementación integral en Salesforce Agentforce.

1. ESPECIFICACIONES GENERALES Y REGLAS DE ORO
---------------------------------------------
- Autenticación: Header 'X-API-Key' obligatorio en todas las peticiones.
- Moneda: Siempre Pesos Mexicanos (MXN).
- Método de Pago: Exclusivamente Tarjeta de Crédito.
- Vigencia: Las cotizaciones y enlaces de pago expiran en 24 horas.
- Rate Limit: 60 solicitudes por minuto. Implementar manejo de error 429.
- Timeouts: Por defecto 30,000ms (30 segundos).

2. FLUJO TRANSACCIONAL COMPLETO (6 PASOS)
-----------------------------------------

PASO 1: Identificación Jerárquica del Vehículo
- Funciones: Navegación guiada por catálogo.
- Endpoints: 
    * GET /bot/catalog/years (Años disponibles)
    * GET /bot/catalog/brands?year={año}
    * GET /bot/catalog/models?year={año}&brandId={idMarca}
    * GET /bot/catalog/versions?year={año}&modelId={idModelo}
- Objetivo: Obtener el 'vehicleVersionId'.

PASO 2: Generación de Cotización (Síncrona y Streaming)
- Endpoint Síncrono: POST /bot/quotation
- Endpoint Streaming (SSE): GET /bot/quotation/:id/stream
- Datos Requeridos:
    * vehicleVersionId (ID obtenido en Paso 1)
    * insured.name (Nombre completo del conductor)
    * insured.phone (Formato +525512345678)
    * insured.zipCode (5 dígitos)
    * insured.birthDate (ISO 8601: AAAA-MM-DD)
    * insured.gender (M o F)
    * coverage (AMPLIA, LIMITADA o RC)
- Datos Opcionales: insured.email, insurers (lista específica), timeout.
- Regla: El bot debe filtrar aseguradoras con error y mostrar solo las exitosas.

PASO 3: Selección de Plan y Captura Dinámica
- Endpoint: POST /bot/quotation/:quotationId/select
- Input: Nombre de la aseguradora (HDI, ANA, GNP, QUALITAS, CHUBB).
- FUNCIONALIDAD CRÍTICA (requiredFields): 
    * La API devuelve una lista dinámica de campos según la aseguradora.
    * El agente DEBE procesar esta lista para preguntar: Tipo (string, date, select), Required (true/false), y presentar Options en caso de ser 'select'.
    * Campos comunes: RFC, VIN (serie), Calle, Ocupación, Estado Civil.

PASO 4: Emisión de Póliza y Validaciones de Datos
- Endpoint: POST /bot/quotation/:quotationId/insured
- Reglas de Formateo Obligatorias:
    * fullName: SOLO nombre(s) de pila (Sin apellidos).
    * paternalSurname y maternalSurname: Apellidos por separado.
    * email: Obligatorio para ANA, GNP, Qualitas y Chubb.
    * maritalStatus: Formato de una sola letra (C, D, S, U, V).
- Conversión Automática: La API convierte gender y maritalStatus al formato interno de cada aseguradora.

PASO 5: Entrega del Enlace de Pago
- Endpoint: GET /bot/policy/:policyId/payment-link
- Acción: Entregar 'paymentUrl' al usuario.
- Información de soporte: Informar monto total y fecha de expiración.

PASO 6: Verificación de Estado
- Endpoint: GET /bot/policy/:policyId
- Estados a monitorear: 'pending' (esperando pago), 'approved' (éxito), 'refused' (rechazo).

3. FUNCIONES AUXILIARES Y DE CONSULTA
-------------------------------------
- Listar Aseguradoras: GET /bot/insurers (Verificar disponibilidad antes de iniciar).
- Listar Coberturas: GET /bot/coverages (Para mostrar opciones al usuario).
- Opciones de Pago: GET /bot/installment-options?quotationId={id}
    * Regla Quálitas: Solo devuelve "contado". El usuario elige meses en el portal de pago.
    * Otras: Devuelve monto por parcela y moneda.

4. SEGUIMIENTO INTERNO (DASHBOARD/VENTAS)
-----------------------------------------
- Listado de Cotizaciones: GET /bot/quotation/status (Paginado).
- Estado Específico: GET /bot/quotation/:id/status.
- Etapas de Rastreo: pending_quotes, quotes_received, insurer_selected, policy_pending, policy_approved, policy_refused.

6. MEJORA: GENERACIÓN DE PDF DE COTIZACIÓN (EXTERNO A API)
-------------------------------------------------------
- Objetivo: Ofrecer un documento descargable al cliente con la comparativa.
- Implementación: 
    * Crear Visualforce Page en Salesforce para diseño de cotización.
    * Acción Invocable que convierta la respuesta JSON de la API en un registro de Salesforce.
    * Generar link de descarga público para el usuario desde el chat de Agentforce.

7. ESTRATEGIA DE IMPLEMENTACIÓN EN AGENTFORCE
---------------------------------------------
FASE 1: Capa de Datos (Named Credentials & External Services) -> YA INICIADO.
FASE 2: Capa de Lógica (Apex Invocable Actions).
    * Crear Acción "Ollin_ProcessRequiredFields": Para transformar el JSON dinámico de campos en preguntas que el agente pueda entender.
FASE 3: Capa de Conversación (Agent Instructions).
    * Configurar el System Prompt con las validaciones de formato (Nombre de pila vs Apellidos, Formato de fecha).
FASE 4: Capa de Excepción.
    * Manejo de reintentos para error 429 y mensajes amigables si el VIN es inválido (error 422).
